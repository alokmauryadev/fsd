Hey! So today we're going deeper into making things move on a webpage using CSS. It's super fun! We'll look at the two main ways: Transitions and Keyframe Animations, but this time, we'll explore all the cool little details.

1. Transitions: The Smooth Change

Transitions are for simple, two-step animations. Something changes from a start state to an end state, usually triggered by something like a hover. The browser smoothly fills in the gaps.

The main ingredients are:

  • Property: *What* are we animating? (e.g., `background-color`, `transform`)
  • Duration: *How long* should it take? (e.g., `0.5s`)
  • Timing Function: *How* does it animate? (the "speed curve", e.g., `ease-in-out`)
  • Delay: *How long* to wait before starting? (e.g., `0.2s`)

transition.css 
button {
    background-color: steelblue;
    transform: scale(1);
    /* Shorthand: property duration timing-function delay */
    transition: all 0.4s ease-in-out 0s;
}

button:hover {
    background-color: tomato;
    transform: scale(1.1);
}

2. Keyframes: The Movie Scene

Keyframes are for more complex animations where you need full control. You can define multiple steps or stages using percentages, from 0% (start) to 100% (end).

Key properties for `animation`:

  • Iteration Count: How many times to repeat? (`infinite` or a number)
  • Direction: Play forwards, backwards, or both? (`normal`, `reverse`, `alternate`)
  • Fill Mode: What styles apply before/after? (`forwards` is super useful to make it stick at the end!)

keyframes.css 
/* 1. Define the animation steps with percentages */
@keyframes pulse {
    0%   { transform: scale(1); }
    50%  { transform: scale(1.1); }
    100% { transform: scale(1); }
}

/* 2. Apply it with all the cool options */
.pulse-button {
    /* name duration timing-function delay iteration-count direction fill-mode */
    animation: pulse 2s ease-in-out infinite;
}

Quick Tips & Good Habits

A couple of things to remember to make your animations awesome!

  • Performance is Key: Animating `transform` (scale, rotate, translate) and `opacity` is super fast! Try to stick to these. Animating things like `width`, `height`, or `margin` can sometimes be slow because it makes the browser re-calculate the whole page layout.
  • Transitions vs. Keyframes:
    • Use Transitions for simple UI feedback (hover effects, opening menus).
    • Use Keyframes for complex, continuous, or attention-grabbing animations (like loaders, notifications, or fun background effects).
